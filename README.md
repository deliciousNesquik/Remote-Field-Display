## <img width="50" alt="Снимок экрана 2025-01-08 в 5 42 10 PM" src="https://github.com/user-attachments/assets/cc38ae1e-2c34-4517-a70d-c954e81564c5" /> Remote Field Display (RFD)
Программный модуль с собственным интерфейсом созданный для модульной системы Genesis. 

RFD - Монитор Бурильщика, ПО визулизирующее данные от буровой установки и помогающее отслеживать состояние бурения
Назначение приложения
Приложение RFD предназначено для мониторинга и анализа работы буровой установки. Его основные функции включают:
1. Получение данных с буровой установки через датчик давления, который фиксирует гидравлические толчки.
2. Преобразование данных с помощью ПО Genesis в числовые значения, понятные для анализа.
3. Передачу данных в приложение RFD и их визуализацию для удобства пользователя.

> [!IMPORTANT]
> Приложение предоставляет буровикам и супервайзерам интуитивно понятный интерфейс для отслеживания текущих параметров бурения, состояний оборудования и выполнения анализа в реальном времени. Это позволяет > > оперативно принимать решения и повышает общую эффективность работы буровой установки.

 
### Разработка
Кроссплатформенное приложение, в первую очередь разрабатываемое для импортозамещения придожения разработанного для Windows

Технология используемая для разработки интерфейса:

Avalonia UI — кроссплатформенная платформа пользовательского интерфейса на основе XAML, основанная на WPF/UWP и распространяемая под лицензией MIT. 

Приложение придерживается паттерну программирования MVVM, что позволяет интерфейсу и логике быть независимыми друг от друга, а также создавать легкомасштабируемые приложения

### Интерфейс приложения
#### Общий вид приложения
Приложение имеет верхнюю кастомную панель управления, которая включает следующие элементы (слева направо):
1. Иконка приложения.
2. Название: "RFD Монитор бурильщика".
3. Кнопки управления: свернуть, развернуть на весь экран, закрыть приложение.

> [!IMPORTANT]
> Все элементы панели выполнены с использованием векторной графики в формате SVG, что обеспечивает их высокое качество при масштабировании.

#### Панель меню
Ниже панели управления расположена панель меню, содержащая следующие пункты:
1. Подключение.
2. Настройки.
3. Окна.
4. Помощь.

В правом углу панели меню отображается статус подключения:
IP (Адрес): показывает текущий IP-адрес сервера.
> [!NOTE]
> При отсутствии подключения отображается сообщение "Нет подключения".

#### Главное пространство интерфейса
Центральная часть интерфейса содержит пять функциональных блоков:
1. Мишень - Круговой элемент, отображающий направление бурения и дополнительную информацию.

2. Параметры бурения - Содержит два ключевых параметра:
 - Магнитное склонение.
 - Смещение инструмента.

3. Информация о бурении - Блок с подробной информацией о процессе бурения, представленной в текстовой и числовой форме.
4. Статусы бурения - Отображает текущее состояние частей буровой установки:
 - Зелёный: нормальная работа.
 - Красный: неисправность.

5. Состояния бурения - Показывает состояние различных деталей буровой установки. Символы:
 - +: в норме.
 - -: неисправность.

Блоки расположены в следующем порядке (слева направо, сверху вниз):
1. Мишень.
2. Параметры бурения.
3. Информация о бурении.
4. Статусы бурения.
5. Состояния бурения.

Все элементы интерфейса связаны через ViewModel, что обеспечивает обновление данных в реальном времени.
Функционал меню

1. Подключение
   
  1.1 Автоматическое подключение:
  - Открывает окно с процессом подключения.
  - Включает индикатор выполнения (вращающийся круг).
  - Предоставляет подсказки и возможные решения при сбоях.
  - Имеет кнопку "Отмена" для прерывания процесса.

  1.2. Ручное подключение:
  -Открывает окно с полем для ввода IP-адреса.
  - Предоставляет инструкции по заполнению поля.
  - Кнопки: "Подключиться" и "Отмена".
> [!NOTE]
> При успешном подключении кнопки блокируются для предотвращения ошибок.

  1.3 Отключение:
  - Разрывает соединение с сервером.
  - Удаляет временные данные.

2. Настройки
   - Позволяет настроить параметры приложения:
   - Выбор языка интерфейса.
   - Выбор темы оформления.

> [!NOTE]
> В будущем — добавление других параметров настройки.

3. Окна - Позволяет управлять видимостью функциональных блоков интерфейса:
 - Мишень.
 - Параметры.
 - Информация.
 - Статусы.
 - Состояния.

> [!IMPORTANT]
> Если пользователь скрывает все элементы на одной стороне, оставшиеся блоки автоматически перераспределяются по рабочей области.

4. Помощь - Содержит следующие разделы:
 - Руководство пользователя — предоставляет справочную информацию по работе с приложением.
 - О приложении — отображает сведения о версии и разработчике.
 
#### Заключение
Интерфейс приложения RFD разработан с целью обеспечения:
- Интуитивной понятности для конечного пользователя.
- Простоты настройки рабочего пространства.
- Гибкости и удобства для дальнейшего масштабирования и модернизации.

> [!IMPORTANT]
> Использование модульной структуры и технологий привязки данных гарантирует актуальность отображаемой информации и упрощает процесс разработки.
